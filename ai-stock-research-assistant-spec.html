产品与技术规格

A股 AI 研究助手 MVP 规格

版本 1.1 更新于 2026-07-14 OpenBB + Qlib + LightGBM 沪深300股票池

状态：Ready for implementation
版本：1.1
日期：2026-07-14
范围：个人使用、仅沪深300成分股、OpenBB统一数据网关、免费近实时数据、无自动交易

1. 背景#

个人投资研究通常分散在行情网站、公告页面、新闻搜索、技术指标工具和手工笔记中。单独展示“明天涨还是跌”既难验证，也容易让用户误把低置信度判断当成事实。

本产品将行情、公告、新闻、异动解释、概率预测、历史相似行情和预测成绩单放在一个本地优先的研究界面中。预测只是一个可审计模块；所有结论必须显示数据截止时间、证据来源、模型版本和历史表现。

1.1 已验证的当前状态#

截至 2026-07-14，工作目录中没有现有项目文件，且不是 Git 仓库。本规格按绿地项目设计，不假设已有数据库、前端或部署环境。

1.2 目标用户#

  • 单个A股个人研究者。
  • 从沪深300当前成分股中建立自选股，不进行系统自动下单。
  • 需要快速回答“发生了什么、为什么、风险在哪、历史上类似情况后来怎样”。

1.3 产品目标#

  1. 在一个界面查看自选股近实时行情、数据新鲜度和关键变化。
  2. 将公告和新闻转换为带来源的结构化影响分析。
  3. 解释放量、跳空、急涨急跌和相对大盘异动。
  4. 输出今日收盘与未来5个交易日的概率、收益区间和置信度。
  5. 展示历史相似状态及其后续真实表现。
  6. 永久保存每次预测并自动结算，公开真实命中率与误差。

1.4 非目标#

  • 不自动下单，不连接实盘交易账户。
  • 不输出“必须买入”“必须卖出”等指令。
  • 第一版不支持港股、美股、基金、期货、期权或加密资产。
  • 第一版不允许新增沪深300之外的A股；历史成分股仅用于无幸存者偏差训练和历史记录。
  • 不承诺收益或最低预测准确率。
  • 不做社交、跟单、社区荐股、组合再平衡和推送通知。
  • 不训练或微调大型语言模型。
  • 不把多个使用相同数据源的 Agent 伪装成独立共识。

3. 核心用户流程#

3.1 添加自选股#

  1. 用户输入 600519 或公司名称。
  2. 系统只返回有效、当前上市且在查询日属于沪深300的证券。
  3. 用户添加后，系统回补至少3年日线和可获得的分钟线。
  4. 数据未回补完成时显示进度，不显示未经训练的预测。
  5. 股票被调出沪深300后保留历史页面和既有预测，标记“已调出沪深300”，停止生成新预测且禁止重新添加。

3.2 查看股票研究页#

  1. 页面顶部显示最新价、涨跌幅、行情时间、数据源和新鲜度。
  2. 页面依次显示异动摘要、预测、公告新闻、历史相似行情和模型成绩。
  3. 每条 AI 结论必须包含可点击证据；没有证据时显示“原因未知”。
  4. 数据超过180秒未更新时，整页显示“行情可能已过期”，禁止把旧行情标记为实时。

3.3 查看预测#

  1. 今日预测最早在交易日 09:45 生成，目标为当日收盘相对昨收的收益率。
  2. 一周预测目标为从预测时点价格到第5个后续交易日收盘的累计收益率。
  3. 输出上涨概率、预期收益、20%/80%分位区间、置信度、数据截止时间和模型版本。
  4. 用户可展开查看滚动验证样本数、方向命中率、MAE、Brier Score 和基准表现。
  5. 模型没有统计优势时仍可显示结果，但必须标记“未优于基准”。

3.4 预测结算#

  1. 今日预测在当日收盘数据确认后结算。
  2. 一周预测在第5个后续交易日收盘后结算。
  3. 原始预测不可覆盖；模型更新必须创建新的模型版本。
  4. 成绩单同时展示全部历史、最近20次和最近100次指标。

4.1 技术选型#

层 选择 原因
统一数据网关 OpenBB Platform 4.7.2 统一Provider、数据Schema和内部REST接口，便于后续扩展
行情与新闻 自定义 OpenBB AKShare Provider，akshare==1.18.64 免费、A股覆盖广；通过OpenBB扩展隔离接口变化
公告 自定义 OpenBB 中国法定披露 Provider 巨潮资讯、上交所和深交所原文为权威来源
股票池 自定义 OpenBB CSI300 Provider 当前成分用于选择，历史成分快照用于训练
量化研究 Microsoft Qlib + LightGBM 支持特征、训练、滚动回测和模型版本化
API Python 3.12+、FastAPI、Pydantic 与量化生态一致，接口契约明确
后台任务 APScheduler 独立 worker 进程 个人MVP不引入 Redis/Celery
数据库 PostgreSQL 16+ 支持时序索引、JSONB证据和事务
前端 Next.js、TypeScript 支持本地Web界面和清晰的数据状态管理
Agent OpenAI-compatible provider；本地 Ollama 可选 模型供应商可替换，默认不绑定商业API
部署 Docker Compose，仅绑定 127.0.0.1 单机个人使用，降低暴露面

4.2 设计约束#

  • 所有外部数据源必须通过自定义OpenBB Provider和 OpenBBGateway 访问，业务代码不得直接调用第三方 URL。
  • 应用业务代码只调用内部OpenBB REST；只有自定义OpenBB Provider可以访问AKShare、巨潮、交易所和中证指数来源。
  • OpenBB不保存业务状态，不替代PostgreSQL、预测账本或Qlib；外部数据必须先规范化并落库后才能进入模型和Agent。
  • OpenBB采用AGPL-3.0，MVP只允许本地个人使用且不得公开托管；公开或商业发布前必须单独完成许可证和底层数据授权审查。
  • 行情记录必须保存 source、source_url 和 observed_at；文档记录另存 published_at。published_at 不适用于报价快照。
  • 预测训练和回测必须使用 point-in-time 数据，禁止使用预测时点之后发布的数据。
  • Agent只能引用检索到的文档，不得生成没有证据的公司事件。
  • 定量概率来自量化模型或校准器，不来自大语言模型文字判断。

5.2 OpenBB 网关接口#

class OpenBBGateway(Protocol):
    async def get_universe_members(self, universe: Literal["CSI300"], as_of: date) -> list[UniverseMemberRecord]: ...
    async def search_instruments(self, universe: Literal["CSI300"], query: str, as_of: date, limit: int) -> list[InstrumentRecord]: ...
    async def get_quotes(self, symbols: list[str], as_of: datetime) -> list[QuoteRecord]: ...
    async def get_bars(self, symbol: str, timeframe: Literal["5m", "1d"], start: datetime, end: datetime) -> list[BarRecord]: ...
    async def get_announcements(self, symbol: str, start: datetime, end: datetime) -> list[DocumentRecord]: ...
    async def get_news(self, symbol: str, start: datetime, end: datetime) -> list[DocumentRecord]: ...

OpenBB固定使用 openbb==4.7.2，自定义扩展版本与应用版本同步。AKShare扩展固定使用 akshare==1.18.64，只允许调用 stock_zh_a_spot_em、stock_zh_a_hist、stock_zh_a_hist_min_em 和 stock_news_em。法定披露扩展只返回巨潮、上交所或深交所原文；CSI300扩展以中证指数官方成分及调整公告为权威来源。参数映射、返回列、Provider命令、样例响应和OpenBB REST路由写入 docs/data-sources.md 并由契约测试锁定。

OpenBB当前没有官方A股Provider，因此上述三个扩展属于本项目交付范围。MVP不做静默备用源：主源失败时进入 stale/unavailable 状态，避免不同口径数据混合。

免费公开数据仅限个人研究，不得重新分发。实现前必须在 docs/data-sources.md 记录每个上游来源的使用条款、访问日期、频率限制和延迟说明；缺少该记录时发布检查失败。

6. 数据模型#

以下为逻辑SQL契约；迁移工具使用 Alembic。

CREATE TABLE instruments (
  symbol          varchar(12) PRIMARY KEY,
  exchange        varchar(8) NOT NULL CHECK (exchange IN ('SSE', 'SZSE')),
  name            varchar(80) NOT NULL,
  industry        varchar(120),
  listed_at       date,
  active          boolean NOT NULL DEFAULT true,
  updated_at      timestamptz NOT NULL
);

CREATE TABLE universes (
  code              varchar(16) PRIMARY KEY,
  name              varchar(80) NOT NULL,
  benchmark_symbol  varchar(12) NOT NULL,
  source            varchar(40) NOT NULL,
  source_url        text NOT NULL,
  snapshot_at       timestamptz NOT NULL
);

CREATE TABLE universe_memberships (
  universe_code    varchar(16) NOT NULL REFERENCES universes(code),
  symbol           varchar(12) NOT NULL REFERENCES instruments(symbol),
  effective_from   date NOT NULL,
  effective_to     date,
  source           varchar(40) NOT NULL,
  source_url       text NOT NULL,
  observed_at      timestamptz NOT NULL,
  PRIMARY KEY (universe_code, symbol, effective_from),
  CHECK (effective_to IS NULL OR effective_to >= effective_from)
);

CREATE TABLE watchlist_items (
  id              bigserial PRIMARY KEY,
  symbol          varchar(12) NOT NULL REFERENCES instruments(symbol),
  universe_code   varchar(16) NOT NULL DEFAULT 'CSI300' REFERENCES universes(code),
  display_order   integer NOT NULL DEFAULT 0,
  created_at      timestamptz NOT NULL DEFAULT now(),
  UNIQUE (symbol)
);

CREATE TABLE quotes (
  symbol          varchar(12) NOT NULL REFERENCES instruments(symbol),
  observed_at     timestamptz NOT NULL,
  price           numeric(18,4) NOT NULL,
  previous_close  numeric(18,4) NOT NULL,
  open            numeric(18,4),
  high            numeric(18,4),
  low             numeric(18,4),
  volume          numeric(24,4),
  amount          numeric(24,4),
  volume_ratio    numeric(12,4),
  source          varchar(40) NOT NULL,
  source_url      text,
  raw_payload     jsonb NOT NULL,
  PRIMARY KEY (symbol, observed_at)
);

CREATE TABLE bars (
  symbol          varchar(12) NOT NULL REFERENCES instruments(symbol),
  timeframe       varchar(8) NOT NULL CHECK (timeframe IN ('5m', '1d')),
  bar_time        timestamptz NOT NULL,
  open            numeric(18,4) NOT NULL,
  high            numeric(18,4) NOT NULL,
  low             numeric(18,4) NOT NULL,
  close           numeric(18,4) NOT NULL,
  volume          numeric(24,4) NOT NULL,
  amount          numeric(24,4),
  adjustment      varchar(8) NOT NULL DEFAULT 'qfq',
  source          varchar(40) NOT NULL,
  source_url      text,
  observed_at     timestamptz NOT NULL,
  PRIMARY KEY (symbol, timeframe, bar_time)
);

CREATE TABLE documents (
  id              uuid PRIMARY KEY,
  symbol          varchar(12) REFERENCES instruments(symbol),
  document_type   varchar(16) NOT NULL CHECK (document_type IN ('announcement', 'news')),
  title           text NOT NULL,
  body_text       text,
  source          varchar(80) NOT NULL,
  source_url      text NOT NULL,
  published_at    timestamptz NOT NULL,
  observed_at     timestamptz NOT NULL,
  content_hash    char(64) NOT NULL UNIQUE
);

CREATE TABLE analyses (
  id              uuid PRIMARY KEY,
  symbol          varchar(12) NOT NULL REFERENCES instruments(symbol),
  analysis_type   varchar(24) NOT NULL CHECK (analysis_type IN ('document', 'anomaly', 'daily_brief')),
  direction       varchar(12) CHECK (direction IN ('positive', 'negative', 'neutral', 'unknown')),
  horizon         varchar(16) CHECK (horizon IN ('intraday', 'short', 'medium', 'unknown')),
  confidence      numeric(5,4) CHECK (confidence BETWEEN 0 AND 1),
  summary         text NOT NULL,
  evidence        jsonb NOT NULL CHECK (jsonb_typeof(evidence) = 'array'),
  model_provider  varchar(80),
  model_name      varchar(120),
  data_cutoff     timestamptz NOT NULL,
  created_at      timestamptz NOT NULL DEFAULT now()
);

CREATE TABLE jobs (
  id                uuid PRIMARY KEY,
  job_type          varchar(32) NOT NULL CHECK (job_type IN ('instrument_backfill', 'analysis_refresh', 'prediction', 'settlement')),
  symbol            varchar(12) REFERENCES instruments(symbol),
  status            varchar(12) NOT NULL CHECK (status IN ('queued', 'running', 'succeeded', 'failed')),
  completed_steps   integer NOT NULL DEFAULT 0 CHECK (completed_steps >= 0),
  total_steps       integer NOT NULL CHECK (total_steps > 0),
  current_step      varchar(40),
  warnings          jsonb NOT NULL DEFAULT '[]'::jsonb CHECK (jsonb_typeof(warnings) = 'array'),
  error_code        varchar(40),
  error_message     text,
  idempotency_key   varchar(160) NOT NULL UNIQUE,
  created_at        timestamptz NOT NULL DEFAULT now(),
  started_at        timestamptz,
  finished_at       timestamptz,
  updated_at        timestamptz NOT NULL DEFAULT now(),
  CHECK (completed_steps <= total_steps)
);

CREATE TABLE model_versions (
  id                uuid PRIMARY KEY,
  model_key         varchar(40) NOT NULL,
  version           varchar(40) NOT NULL,
  target_horizon    varchar(16) NOT NULL CHECK (target_horizon IN ('today_close', 'next_5d')),
  feature_schema    jsonb NOT NULL,
  train_start       date NOT NULL,
  train_end         date NOT NULL,
  validation_metrics jsonb NOT NULL,
  artifact_uri      text NOT NULL,
  status            varchar(12) NOT NULL CHECK (status IN ('candidate', 'active', 'retired')),
  created_at        timestamptz NOT NULL DEFAULT now(),
  UNIQUE (model_key, version)
);

CREATE TABLE predictions (
  id                uuid PRIMARY KEY,
  symbol            varchar(12) NOT NULL REFERENCES instruments(symbol),
  model_version_id  uuid NOT NULL REFERENCES model_versions(id),
  horizon           varchar(16) NOT NULL CHECK (horizon IN ('today_close', 'next_5d')),
  as_of              timestamptz NOT NULL,
  target_at          timestamptz NOT NULL,
  reference_price    numeric(18,4) NOT NULL,
  probability_up     numeric(5,4) NOT NULL CHECK (probability_up BETWEEN 0 AND 1),
  expected_return    numeric(12,8) NOT NULL,
  lower_return       numeric(12,8) NOT NULL,
  upper_return       numeric(12,8) NOT NULL,
  confidence_label   varchar(8) NOT NULL CHECK (confidence_label IN ('low', 'medium', 'high')),
  data_cutoff        timestamptz NOT NULL,
  features_snapshot  jsonb NOT NULL,
  created_at         timestamptz NOT NULL DEFAULT now(),
  UNIQUE (symbol, model_version_id, horizon, as_of)
);

CREATE TABLE prediction_outcomes (
  prediction_id      uuid PRIMARY KEY REFERENCES predictions(id),
  actual_price        numeric(18,4) NOT NULL,
  actual_return       numeric(12,8) NOT NULL,
  direction_correct   boolean NOT NULL,
  absolute_error      numeric(12,8) NOT NULL,
  settled_at          timestamptz NOT NULL
);

CREATE INDEX bars_symbol_time_idx ON bars (symbol, timeframe, bar_time DESC);
CREATE INDEX universe_membership_active_idx ON universe_memberships (universe_code, effective_from, effective_to, symbol);
CREATE INDEX documents_symbol_published_idx ON documents (symbol, published_at DESC);
CREATE INDEX predictions_symbol_asof_idx ON predictions (symbol, horizon, as_of DESC);

7. API 契约#

API前缀统一为 /api/v1。MVP无登录功能，只允许通过绑定在 127.0.0.1 的本地服务访问。所有响应都包含 request_id；列表使用游标分页，limit 默认20、最大100。游标是以下JSON的无填充Base64URL编码：{"v":1,"sort":"published_at","value":"2026-07-14T00:00:00Z","id":"uuid"}；无效版本或字段返回 400 INVALID_ARGUMENT。

Pydantic/OpenAPI是DTO的唯一来源，必须定义 InstrumentDTO、WatchlistItemDTO、QuoteDTO、DocumentDTO、EvidenceDTO、AnalysisDTO、PredictionDTO、ScorecardDTO、AnalogDTO 和 JobDTO。数据库模型不得直接序列化给客户端；CI比较生成的 openapi.json，未审查的契约变化会失败。

列表响应：

{
  "data": [],
  "page": {"next_cursor": null, "has_more": false},
  "request_id": "uuid"
}

错误响应统一为：

{
  "error": {
    "code": "DATA_STALE",
    "message": "Quote is older than 180 seconds",
    "request_id": "uuid"
  }
}
HTTP状态 错误码 语义
400 INVALID_ARGUMENT 代码、分页或时间参数无效
404 INSTRUMENT_NOT_FOUND 证券不存在或不在MVP市场范围
409 NOT_CURRENT_UNIVERSE_MEMBER 股票不是查询日沪深300当前成分股
409 DUPLICATE_WATCHLIST_ITEM 重复添加自选股
422 INSUFFICIENT_DATA 历史样本不足，无法生成结果
424 PROVIDER_UNAVAILABLE 上游数据源失败且无可用结果
503 MODEL_UNAVAILABLE 没有可用模型版本

行情过期但仍有最后值时返回 200，freshness=stale 并附 age_seconds；从未取得行情时返回 424。预测不存在但回补仍在进行时返回 202 和作业状态；确认样本不足时返回 422。

  • GET /watchlist

8. 数据采集与调度#

所有时间按 Asia/Shanghai 处理，交易日历以交易所日历为准。

作业 交易日调度 失败处理
沪深300成分同步 每日07:30、18:30 保留上一快照并标记同步失败，不覆盖历史有效期
自选股报价 09:25-11:30、13:00-15:00 每15秒 指数退避；180秒后标记 stale
5分钟K线 09:35-11:30、13:05-15:05 每60秒 按主键幂等补写
日线 15:10、18:00各一次 对账后覆盖同源未确认记录
公告 交易时段每5分钟，其他时段每小时 按内容哈希去重
新闻 交易时段每10分钟，其他时段每2小时 按URL和内容哈希去重
今日预测 09:45起每15分钟，最后一次14:45 模型不可用则显示 unavailable
一周预测 09:45、11:30、15:20 保留全部版本，不覆盖
预测结算 15:20及次日08:30补偿 幂等；交易日顺延

数据源连续失败3次后进入降级状态，界面展示具体失败源和最后成功时间。不得静默使用缓存冒充新数据。

9. 量化预测设计#

9.1 预测目标#

  • today_close：当日收盘价 / 昨日收盘价 - 1。
  • next_5d：第5个后续交易日收盘价 / 预测参考价 - 1。
  • 方向标签：目标收益率大于0记为上涨，否则记为非上涨。

9.2 特征#

  • 价格动量：1/2/5/10/20/60期收益。
  • 趋势：价格相对5/10/20/60期均线距离。
  • 波动：5/20期已实现波动、ATR、日内振幅。
  • 成交：成交量相对20期均值、换手率、量比。
  • 市场：沪深300和上证指数同期收益、相对强弱。
  • 今日模型专用：开盘缺口、09:45后当日收益、早盘区间和早盘成交占比。
  • 事件特征只允许使用 published_at <= data_cutoff 的文档。

9.3 训练和验证#

  1. 日线至少回补3年；数据不足时不启用一周模型。
  2. 今日模型至少需要120个有效交易日；早盘分钟特征不足时退化为开盘模型并标记原因。
  3. 使用按时间顺序的 expanding-window walk-forward 验证，禁止随机切分。
  4. 训练、验证、测试时间段不得重叠；归一化参数只在训练窗口拟合。
  5. 概率使用验证集校准，保存未校准和已校准指标。
  6. 基准包括：恒定上涨概率、历史均值收益和沪深300方向。
  7. 每个模型版本保存特征Schema、代码提交SHA、数据范围、依赖锁文件哈希和指标。
  8. 训练样本按每个交易日当时有效的CSI300历史成员生成，禁止用当前300只股票回填全部历史。
  9. 线上推理只为 as_of 当日有效CSI300成员生成；调出成员的历史预测继续结算但不产生新预测。

9.3.1 Qlib 数据与产物契约#

  • 日线与分钟线先从PostgreSQL导出为按 instrument、datetime 排序的Parquet快照，再转换为Qlib dataset；快照清单保存行数、最小/最大时间和SHA-256。
  • Qlib instruments 文件由 universe_memberships 的历史有效期生成；每个成员必须保留真实 effective_from/effective_to，避免幸存者偏差。
  • 特征定义位于 config/features/{feature_set_version}.yaml，任何字段、窗口或缺失值策略变化都必须升级版本。
  • LightGBM同时训练回归目标和方向分类目标；最终上涨概率来自方向模型。
  • 概率校准使用时间验证集上的 isotonic regression；验证样本少于200时使用 Platt scaling，并在模型卡中记录降级。
  • 模型产物保存到只读目录 artifacts/models/{model_key}/{version}/，至少包含模型、校准器、特征Schema、数据清单、指标和模型卡。
  • artifact_uri 使用 file:///models/{model_key}/{version}，容器内只读挂载。
  • better_than_baseline=true 当且仅当候选模型的 Brier Score 和 MAE 均严格优于相同测试窗口上的恒定概率与历史均值基准。
  • 每日计算特征PSI；任一关键特征最近20日 PSI 大于0.20标记漂移，大于0.30时停止生成新预测并返回 MODEL_UNAVAILABLE。

9.4 模型发布门槛#

候选模型成为可运行的 active 版本必须同时满足：

  • 无未来数据泄漏测试失败。
  • 验证覆盖不少于120个日预测或60个周预测。
  • Brier Score、MAE、方向准确率及对应基准均已计算且为有限数值。
  • 输出区间覆盖率、方向准确率和样本数完整可见。

不要求模型达到预设收益率。未优于基准的模型仍可成为 active 研究模型，但 better_than_baseline=false 且置信度只能为 low；只有同时满足第9.3.1节优势判定的模型才能显示 medium 或 high。candidate 状态永远不对API提供预测。

9.5 置信度#

  • high：better_than_baseline=true、验证样本不少于最低门槛的2倍、校准合格且所有关键特征PSI不超过0.10。
  • medium：better_than_baseline=true、达到最低验证样本门槛、校准合格且所有关键特征PSI不超过0.20。
  • low：better_than_baseline=false、数据降级或任一关键特征PSI超过0.20；低于最低样本门槛的模型不得成为 active。

10. 历史相似行情#

  1. 使用与当前模型一致的 point-in-time 特征。
  2. 用训练集均值和标准差标准化，不读取目标期数据。
  3. 默认返回距离最近的10个历史状态。
  4. 展示后续1日和5日收益分布，不将相似案例描述为因果关系。
  5. 当有效候选少于30个时关闭该功能并说明样本不足。

11. 分析 Agent#

Agent是证据整理器，不是数值预测器。

11.1 工具#

  • get_quote_snapshot(symbol, as_of)
  • get_recent_bars(symbol, timeframe, limit, as_of)
  • get_documents(symbol, start, end, as_of)
  • get_benchmark_snapshot(symbol, as_of)
  • get_model_prediction(symbol, horizon, as_of)

11.2 固定输出Schema#

{
  "summary": "string",
  "direction": "positive|negative|neutral|unknown",
  "horizon": "intraday|short|medium|unknown",
  "confidence": 0.0,
  "evidence_ids": ["uuid"],
  "unknowns": ["string"],
  "risk_flags": ["string"]
}

11.3 约束#

  • evidence_ids 为空时，方向必须为 unknown。
  • evidence_ids 是Agent内部输出。API层必须在同一事务快照中查询对应 documents，展开为第7.3节的 EvidenceDTO；任一ID不存在时整条分析校验失败，不返回部分证据。
  • Agent不得修改量化模型概率。
  • Agent不得推断未披露业绩、内幕信息或实时资金来源。
  • 每个事实句必须能映射到行情字段或文档引用。
  • JSON Schema校验失败时最多重试一次；再次失败则使用模板摘要。

12. 异动检测#

满足任一条件时创建异动事件：

  • 5分钟收益绝对值超过过去60日同时间段的99百分位。
  • 当日成交量进度超过过去20日同时间进度均值的2倍。
  • 股票当日收益与沪深300收益差绝对值超过2个百分点。
  • 开盘缺口绝对值超过过去60日的95百分位。

异动分析先给出确定性量价事实，再检索事件证据。没有匹配公告或新闻时必须写“未找到可验证事件原因”。

13. 前端要求#

13.1 页面#

  • /：自选股表格，支持排序、搜索和数据新鲜度。
  • /stocks/[symbol]：股票研究页。
  • /scorecard：模型和股票维度的预测成绩单。
  • /settings/data-sources：数据源、最后成功时间和模型连接状态。

13.2 必须状态#

  • 首次回补、正常、部分数据缺失、行情过期、数据源失败、模型不可用、无文档、无预测、休市。
  • 任何预测区域都显示“仅供研究，不构成投资建议”。
  • 概率和区间必须同时出现，不允许只显示“看涨/看跌”。
  • 页面不得出现交易按钮、持仓收益诱导或收益承诺。

14. 非功能要求#

14.1 性能#

  • 本地网络条件下，自选股20只以内的首屏API P95小于800毫秒，不含首次数据回补。
  • 单只股票研究页缓存命中时 P95小于1秒。
  • 后台采集不得阻塞API进程。

14.2 可靠性#

  • 所有采集和结算作业幂等。
  • 外部数据写入前进行字段范围、时间和OHLC一致性校验。
  • 数据源故障不得导致进程崩溃或已有历史数据丢失。
  • 每日生成数据库备份，保留7天。

14.3 安全#

  • Docker端口默认只绑定 127.0.0.1。
  • API密钥仅通过环境变量或本地秘密文件注入，不写入数据库日志。
  • Agent获取的数据只限产品数据库，不读取本地任意文件。
  • 外部文档视为不可信内容，不允许其改变系统提示或调用权限。

14.4 可观测性#

  • 每个请求和后台作业生成 request_id 或 job_id。
  • 记录数据源成功率、延迟、旧数据数量、预测生成数量和结算积压。
  • 日志不得包含完整API密钥、用户机器路径或原始提示中的敏感数据。

15. 验收标准#

  1. 用户可以从沪深300当前成分股中添加、删除和重排至少20只证券；非成分股返回 409 NOT_CURRENT_UNIVERSE_MEMBER，重复添加返回明确错误。
  2. 交易时段内，行情成功时页面显示源时间；超过180秒自动显示过期状态。
  3. 休市时明确显示最新交易日和“休市”，不把最新收盘价标成实时。
  4. 公告和新闻按内容哈希去重，同一内容不会重复展示。
  5. 每条AI事实句都能映射到行情字段或包含非空 document_id、source_url、quote 的证据；无证据时方向为 unknown 且包含固定未知原因文本。
  6. 今日预测在09:45前不可用；09:45后包含概率、预期收益、P20/P80区间、模型版本、数据截止时间和置信度。
  7. 一周预测严格指向第5个后续交易日，节假日不会按自然日错误结算。
  8. 任一历史预测创建后不可更新核心预测字段，只能新增结算记录。
  9. 今日和一周预测均可自动结算并计算方向正确性与绝对误差。
  10. 成绩单正确展示全部、最近20次和最近100次的样本数、命中率、MAE与Brier Score。
  11. 回测测试证明任何特征均未读取 data_cutoff 之后的行情或文档。
  12. 模型未优于基准时，界面显示“未优于基准”且置信度为低。
  13. 历史相似行情只使用当时可见特征，并展示真实后续收益。
  14. 外部行情源连续失败时，已有历史页面仍可访问，且状态页显示失败原因和最后成功时间。
  15. 在全新机器上执行 Docker Compose 后，可以完成数据库迁移、加入 600519、回补数据并打开研究页。
  16. 单元、集成和端到端测试全部通过，未出现自动下单或交易账户连接代码。
  17. 指定任一历史日期时，成分接口返回当日真实沪深300成员，不使用当前成分回填历史。
  18. 模拟股票调出指数后，历史页面和既有预测仍可访问，新预测停止且不能重新添加。
  19. 应用层集成测试证明所有外部数据请求都经过OpenBB内部REST，没有直接第三方URL调用。

19. 发布与回滚#

19.1 发布#

  • M1第一步执行 git init 并创建项目初始提交；后续镜像标签所需的 git_sha 来自该仓库。没有提交SHA时发布脚本必须失败。
  • 通过 Docker Compose 发布本地版本；镜像使用不可变标签 app-{git_sha} 和 model-{version}，不得只使用 latest。
  • 数据库迁移在服务启动前执行，失败则不启动新版本。
  • 模型先以 candidate 导入，通过校验后原子切换为 active。
  • 发布包必须包含数据源说明、模型卡、限制和恢复步骤。

19.2 回滚#

  • 应用问题：把 .env 中镜像标签改回上一 git_sha，执行 docker compose pull && docker compose up -d。
  • 数据库问题：每次迁移前执行 pg_dump -Fc -f backups/pre-migrate-{timestamp}.dump。可逆迁移执行 alembic downgrade -1；标记为不可逆的迁移禁止自动发布，只允许先在数据库副本验证后维护窗口执行。恢复命令为 pg_restore --clean --if-exists -d app backups/{file}.dump。
  • 模型问题：将上一模型版本重新标记为 active，保留新模型产生的预测审计记录。
  • 数据源问题：禁用故障Provider并保留旧数据的过期标记，不删除历史记录。

由回滚版本创建的预测继续按其原始 model_version_id 结算，不重算、不删除。旧应用必须能读取新Schema；破坏向后兼容性的Schema变化需要先发布兼容读写版本，再在后续版本删除旧字段。
