## 1. 评审范围与结论

本专题基于三路独立 Agent 评审和主流程交叉验证形成：

- 信息架构：导航、页面层级、术语、用户心智和任务优先级；
- 核心流程：搜索、切换研究范围、添加和移除关注、查看详情、行情与分析状态；
- 视觉与无障碍：桌面端、移动端、宽表、键盘、颜色对比、图表替代内容；
- 页面实测：使用 Playwright 和固定模拟数据检查 1440px、390px、320px 视口，不依赖部署环境或公网数据。

当前产品已经具备以下良好基础：

- 实时行情与历史行情在结构上分离；
- 预测展示概率、区间、数据截止时间和基准对照；
- 分析证据保留来源链接和原文片段；
- 数据异常、回补、模型不可用等状态已有统一组件；
- 页面没有交易入口，并持续展示研究用途声明。

评审时确认的最高风险不是视觉风格，而是以下四项会影响用户判断的体验问题：

1. 切换研究范围时可能出现“新标签配旧列表”；
2. 列表与详情对行情“最新、延迟、过期”的解释不一致；
3. 预测成绩单仍使用旧自选股接口，可能看不到自动研究池股票；
4. 300只股票缺少用于发现研究目标的筛选和排序。

## 2. 产品目标与设计原则

用户打开网站后，应能在较短时间内完成以下任务：

1. 判断当前市场和行情数据是否可信、是否足够新；
2. 从300只股票中快速发现需要关注的异动、公告或待分析对象；
3. 进入详情后先看到关键结论，再按需查看走势、证据和模型验证；
4. 完成添加、刷新、分析或回补操作后，明确知道系统是否受理、当前进度和最终结果；
5. 在手机上完成同样的核心研究流程，而不是只能查看被截断的桌面表格。

设计原则：

- **可信优先**：不同页面不得用不同口径描述同一份数据。
- **研究任务优先**：股票搜索、筛选和列表应早于运维状态和低频操作。
- **真实状态优先**：加载失败、确实为空、尚未生成和样本不足必须分别展示。
- **渐进展示**：先显示行情和研究摘要，再显示技术明细。
- **移动端重排**：窄屏不是简单横向滚动桌面表格，而是重新组织信息层级。
- **状态可恢复**：范围、关键词、排序和页码应保留在URL中，进入详情再返回时不丢失。

## 3. P0：数据可信度与上下文一致性

### 3.1 切换范围时可能显示旧数据

当前 `useApiResource` 在依赖变化时保留旧 `data` 和 `loaded`。页面只在首次加载时显示“加载中”，因此从“沪深300”切换到“我的关注”后，旧的沪深300列表可能暂时出现在新标题下。

相关位置：

- `apps/web/lib/hooks/useApiResource.ts`
- `apps/web/app/page.tsx`

改进要求：

- 请求结果必须和 `scope + query + sort + page` 请求键绑定；
- 切换范围时显示保持版式的骨架屏，或明确显示“正在切换研究范围”；
- 若保留旧数据，标题和筛选上下文也必须保持旧状态，直到新请求完成；
- 旧请求晚于新请求返回时不得覆盖新结果；
- 列表容器在加载时设置 `aria-busy="true"`；
- 股票详情切换代码时同样不得短暂显示上一只股票的数据。

### 3.2 行情新鲜度口径冲突

当前后端已经提供三档 `age_status`：

| 数据年龄 | 状态 |
|---:|---|
| 0至45秒 | 最新 |
| 46至120秒 | 行情可能延迟 |
| 超过120秒 | 可能过期 |

但兼容字段 `freshness` 仍按180秒判断。列表主要读取 `age_status`，详情页的 `resolveQuoteStatus` 主要读取 `freshness`，可能出现：

- 列表显示“行情可能延迟”，详情仍标“实时”；
- 121至180秒时列表显示过期，详情仍显示“3分钟内（新鲜）”；
- 首页把超过120秒的数据计入过期数量，提示却写成“超过180秒”。

相关位置：

- `apps/api/app/services/freshness.py`
- `apps/web/lib/ui-state.ts`
- `apps/web/components/QuoteHeader.tsx`
- `apps/web/components/WatchlistTable.tsx`
- `apps/web/app/page.tsx`

改进要求：

- `age_status + data_age_seconds` 作为全站唯一展示口径；
- 列表、详情、汇总卡和状态提示共用一个状态解析函数；
- 页面显示实际数据年龄，例如“约68秒前获取”，不再统一写“3分钟内”；
- 延迟状态禁止显示“实时”徽标；
- 休市时显示“最近交易时段行情”，不继续使用盘中秒级语义；
- 增加45、46、120、121、180、181秒边界测试。

### 3.3 预测成绩单仍使用旧集合接口

成绩单的股票选择器仍调用旧 `/watchlist`。研究池迁移后，旧接口主要对应额外关注，自动进入研究池的沪深300股票可能无法出现在选择器中。

相关位置：

- `apps/web/app/scorecard/page.tsx`
- `apps/web/lib/api/endpoints.ts`

改进要求：

- 改为读取 `/research-pool?scope=all`；
- 选择项显示“股票名称（代码）”，支持代码和名称搜索；
- 详情页“查看完整成绩单”携带 `symbol` 和 `horizon` 参数；
- 空状态改为“当前研究池暂无可查看的预测记录”，不再提示“请先添加自选股”。

### 3.4 研究池缺少发现目标的工具

002已经定义方向、行情状态、异动、公告、预测和行业筛选，以及成交额、数据年龄、异动强度和分析更新时间排序。当前列表主要只有代码、名称、价格、涨跌幅和新鲜度排序。

改进要求：

- 支持上涨、下跌、平盘；
- 支持最新、延迟、过期、无行情；
- 支持有异动、有重要公告、有预测、等待分析；
- 支持行业筛选；
- 支持涨跌幅、成交额、数据年龄、异动强度、分析更新时间排序；
- 首页摘要数字可点击，并直接形成对应筛选；
- 已生效筛选以可移除标签展示，并提供“清除全部”。

## 4. P1：研究池首页体验

### 4.1 调整首屏顺序

当前默认范围是沪深300，但列表前依次显示四张工作台卡片和“添加额外自选”。用户的主要任务——搜索并打开股票——被推迟。

建议顺序：

```text
页面标题与市场状态
研究范围 + 统一股票搜索 + 添加关注入口
可点击摘要：上涨 / 下跌 / 行情异常 / 待分析 / 重要事件
筛选、排序与结果数量
股票列表
数据源健康等次级信息
```

“添加关注”建议收进“我的关注”范围、右上角抽屉或弹窗，不在沪深300默认页长期占用完整区块。

### 4.2 统一搜索状态

当前表格同时存在：

- 输入时对当前数据做本地过滤；
- 提交后再请求服务端搜索。

切换范围时，父组件清除了服务端关键词，但表格内部输入值不会同步清空，容易出现隐藏的旧筛选。

改进要求：

- 搜索只保留一个受控状态；
- 状态写入URL，例如 `?scope=csi300&q=银行&sort=change_desc&page=1`；
- 输入采用300至500毫秒防抖，或者明确使用“搜索”提交，不同时混用两种语义；
- 显示结果数量、清除按钮和无结果原因；
- 返回列表时恢复原范围、关键词、排序和页码。

### 4.3 精简默认列

桌面端默认列建议为：

| 字段 | 展示要求 |
|---|---|
| 股票 | 名称为主、代码为次，整格可点击 |
| 最新价 | 保留两位小数 |
| 涨跌幅 | 正负号和文字不能只依赖颜色 |
| 成交额 | 使用万、亿单位，精确值放提示或展开区 |
| 行情状态 | 最新、延迟、过期、无行情 + 实际年龄 |
| 研究状态 | 已分析、排队中、失败、暂无触发条件 |
| 操作 | 仅在“我的关注”显示移除等用户操作 |

今开、最高、最低、昨收、成交量和详细时间放入展开行或详情抽屉。系统研究池不需要长期显示“系统研究池”操作列。

### 4.4 改善扫描效率

- 桌面端固定表头和股票列；
- 行悬停和键盘聚焦时提供明显背景；
- 成交量和成交额采用易读单位；
- “顺序”列如果没有用户排序含义则移除；
- 名称和代码都可进入详情；
- 每页数量允许选择25、50、100，并记住用户偏好。

## 5. P1：股票详情体验

### 5.1 改为渐进加载

当前详情页先等待快照、日线、分钟线、异动、预测、文档、分析和相似行情，再读取模型成绩。任何较慢接口都会让顶部行情长期停留在单行“加载中”。

建议拆分为：

1. 快照优先：股票名称、代码、当前行情先显示；
2. 历史行情独立加载；
3. 异动与事件独立加载；
4. 预测独立加载；
5. 相似行情和模型验证低优先级加载；
6. 每个区块拥有自己的加载、失败和重试状态。

### 5.2 区分空状态和错误状态

页面不得把请求失败显示成“没有异动”“没有公告”或“暂无相似行情”。推荐状态：

| 状态 | 示例文案 |
|---|---|
| 加载中 | 正在读取公告与新闻… |
| 加载失败 | 公告数据暂时读取失败，最后成功于10:32。[重试] |
| 确实为空 | 截至10:35，当前没有取得新的公告或新闻。 |
| 尚未生成 | 文档已取得，分析摘要正在排队。 |
| 样本不足 | 当前只有18个有效历史样本，至少需要30个。 |

### 5.3 增加位置感和页内导航

详情顶部增加：

- `研究池 / 贵州茅台 600519` 面包屑；
- 返回筛选结果；
- 上一只、下一只；
- 页内导航：概览、走势、事件、预测、验证。

页面较长时，页内导航可在桌面端粘性显示；移动端使用横向可滚动的紧凑标签栏。

### 5.4 优化首屏内容

行情卡建议补充：

- 涨跌额；
- 今开、最高、最低、昨收；
- 成交量、成交额、换手率；
- 行情时间、获取时间和实际数据年龄；
- 行情延迟或过期时的“获取新行情”。

内部来源标识、模型key和版本等技术信息继续放入折叠区。预测中的P20/P80、MAE和Brier Score应增加一句通俗解释。

## 6. P1：操作闭环与状态反馈

### 6.1 添加关注

添加非沪深300股票成功后：

- 显示“已加入我的关注”；
- 自动切换到“我的关注”，或给出“查看该股票”入口；
- 搜索结果立即标记为“已关注”；
- 展示首次回补状态和下一步；
- 清除旧的添加错误和重复提交状态。

### 6.2 移除关注

当前移除失败复用了添加区错误，用户可能在表格右侧操作后看不到原因。

改进要求：

- 点击后按钮显示“移除中…”；
- 成功显示轻提示并提供撤销；
- 失败信息出现在当前行或全局可见提示中；
- 移除不会删除历史行情、分析和预测时，应在确认文案中说明。

### 6.3 行情刷新

- 首页“刷新”若只读取本地数据，应改名为“重新读取列表”；
- 延迟、过期或无报价时，详情页提供“获取新行情”；
- 明确区分“读取本地最新值”和“请求上游行情”；
- 展示排队、获取中、成功、失败和频率限制状态。

### 6.4 分析与回补

- “等待分析”应说明是排队中、暂未触发还是无需分析；
- 如允许手动触发，提供“生成/更新分析”入口；
- 回补运行期间自动轮询，结束后刷新相关模块；
- 作业查询失败不得静默吞掉；
- 失败时给出重试或重新发起入口；
- 页面离开后再次进入，应能恢复当前作业状态。

## 7. P1：移动端与响应式

### 7.1 修改前实测结果

实施前使用固定模拟数据检查页面：

| 视口 | 观察结果 |
|---:|---|
| 1440px | 研究池宽表仍需要横向滚动，列扫描距离较长 |
| 390px | 表格宽度约为可见容器的4倍，状态和操作隐藏在屏幕右侧 |
| 390px | 第一条股票约在页面纵向807px后出现 |
| 320px | 页面宽度溢出到约357px，添加表单和范围按钮明显拥挤 |

修改前CSS没有专门的媒体查询，主要依赖 `flex-wrap` 和横向滚动，因此只能避免部分元素完全消失，不能形成适合手机的任务流程。

### 7.2 建议断点

#### 小于等于600px

- 主内容左右留白改为12至16px；
- 顶部导航紧凑化，免责声明移动到底部或信息菜单；
- 表单纵向排列，输入框 `min-width: 0; width: 100%`；
- 研究范围和图表周期使用单行横向滚动标签，文字不竖排；
- 工作台摘要压缩为2×2或横向概览；
- 股票列表改为摘要卡片；
- 主要操作触控高度至少44px。

#### 601至960px

- 使用精简表格；
- 隐藏OHLC和精确成交量等次要列；
- 筛选条件进入可展开面板；
- 详情行情字段使用两列网格。

#### 大于960px

- 使用完整精简表格；
- 固定表头和股票列；
- 高级字段通过展开行或列偏好显示。

### 7.3 移动端股票卡片建议

```text
贵州茅台  600519                         >
1,215.04                 +0.33%

行情：最新 · 12秒前
研究：已分析 · 有重要公告
成交额：38.70亿元
```

整张卡片可进入详情。额外关注的移除操作放入更多菜单，避免误触。

## 8. P1/P2：无障碍与视觉规范

### 8.1 颜色对比

当前实算结果：

| 颜色 | 白底对比度 | 风险 |
|---|---:|---|
| 绿色 `#1e8e5a` | 约4.14:1 | 低于普通小字4.5:1要求 |
| 橙色 `#b26a00` | 约4.24:1 | 低于普通小字4.5:1要求 |
| 边框 `#dfe3e8` | 约1.29:1 | 输入框和按钮边界不够清晰 |

改进要求：

- 加深绿色和橙色，并在实际徽标背景上重新计算；
- 表单、按钮、选中态边界达到至少3:1的非文本对比；
- 涨跌、告警和成功不能只依赖颜色，继续同时显示符号和文字。

### 8.2 键盘与焦点

- 全站增加统一的 `:focus-visible`，使用2至3px高对比焦点环；
- 导航链接增加当前页样式和 `aria-current="page"`；
- 页面顶部增加“跳到主要内容”链接；
- 横向滚动区如继续保留，应可聚焦并带区域标签和滚动提示；
- 股票链接扩展到整格或整张卡片，避免16px的小点击目标；
- 弹窗或抽屉必须管理焦点进入、循环和返回。

### 8.3 表格语义

- 研究池、预测历史和模型成绩表增加 `caption` 或 `aria-labelledby`；
- 排序按钮继续维护正确的 `aria-sort`；
- 筛选结果数量和加载完成使用温和的 `aria-live`；
- 危险错误使用 `role="alert"`，普通状态使用 `role="status"`；
- 表头在视觉固定后仍保持正确的DOM表格语义。

### 8.4 图表替代内容

ECharts目前已有趋势摘要和 `role="img"`，但键盘用户无法查询任意日期的精确值。

改进要求：

- 提供“查看数据表”入口；
- 数据表包含日期、收盘价、涨跌额和涨跌幅；
- 图表无障碍描述包含区间、起止值和整体趋势；
- 动画尊重 `prefers-reduced-motion`；
- 如提供下载，明确数据周期、复权口径和更新时间。

## 9. 目标信息架构

## 11. 验收标准

### 11.1 数据与上下文

- 快速切换三个研究范围时，页面不会在新标题下显示旧范围股票。
- 快速切换股票代码时，不会显示上一只股票的行情或预测。
- 列表、详情和首页摘要对同一报价显示完全相同的新鲜度状态。
- 46至120秒的数据不会在任何位置显示为“实时”。
- 成绩单可选择沪深300和我的关注中的全部股票。

### 11.2 研究效率

- 用户可以在不逐页浏览的情况下筛选行情异常、重要事件和等待分析股票。
- 摘要数字可点击并形成对应筛选。
- 搜索、范围、排序和页码可通过URL恢复。
- 返回研究池时保留进入详情前的上下文。
- 桌面端无需横向滚动即可同时看到股票、价格、涨跌、行情状态和研究状态。

### 11.3 状态与操作

- 每个异步区块都有加载、失败、真实为空和重试状态。
- 添加成功后能立即找到新股票并查看回补状态。
- 移除失败在操作位置附近显示，成功后可撤销。
- “重新读取列表”和“获取新行情”是两个明确不同的操作。
- 回补和分析运行期间自动更新，不要求用户反复点击刷新。

### 11.4 响应式

- 320px、390px、768px、1440px下页面不得产生整页横向溢出。
- 390px下，核心搜索和至少第一条股票应在合理的首屏滚动范围内出现。
- 移动端不展示需要四倍屏宽才能操作的桌面表格。
- 输入框、范围标签和图表周期文字不得竖排。
- 主要触控目标至少44×44px。

### 11.5 无障碍

- 普通文字对比度至少4.5:1；大文字至少3:1；控件边界至少3:1。
- 所有交互元素可使用键盘操作，并有清晰的焦点指示。
- 导航正确标注当前页，存在跳到主要内容的入口。
- 表格有可识别标题，动态结果和错误得到适当播报。
- 历史图表存在可通过键盘访问的数据表替代。
- 自动化测试覆盖关键键盘流程，并使用无障碍扫描检查主要页面。

## 12. 边界与保留项

本专题不改变以下产品红线：

- 不增加交易、下单或收益承诺功能；
- 不用历史收盘价填充实时行情；
- 不隐藏数据来源、数据截止时间和模型局限；
- 不因视觉简化而删除预测区间、基准对照或证据来源；
- 不把上游失败展示成正常空数据；
- 不要求本轮进行部署，代码实现后先完成本地可执行的自动化和页面测试。
