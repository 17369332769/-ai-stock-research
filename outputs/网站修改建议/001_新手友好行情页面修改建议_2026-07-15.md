## 1. 问题概述

根据当前页面截图，主要有四个理解障碍：

1. `sina_via_akshare` 是内部技术标识，普通用户无法判断它代表新浪财经还是 AKShare。
2. 折线图没有说明横轴、纵轴和每个点分别代表什么，也没有直接告诉用户这是不是实时价格。
3. “前复权”是专业术语，页面没有解释它为什么存在、会对历史价格产生什么影响。
4. “暂无实时行情”只有结果，没有可执行操作，也没有告诉用户获取一次行情通常需要多久。

本轮改造的重点不是增加更多指标，而是让用户在几秒内回答四个问题：

- 我正在看什么数据？
- 这是不是实时行情？
- 这条线应该怎么读？
- 没有行情时我能做什么？

## 2. 数据源改为中文展示

### 2.1 展示规则

页面主界面只显示中文来源名称，采集工具作为次要说明，内部标识放入“技术信息”折叠区。

| 内部标识 | 主界面中文名称 | 次要说明 |
|---|---|---|
| `sina_via_akshare` | 新浪财经行情 | 通过 AKShare 采集 |
| `eastmoney_via_akshare` | 东方财富行情 | 通过 AKShare 采集 |
| `cninfo` | 巨潮资讯 | 上市公司法定公告 |
| `csindex` | 中证指数 | 沪深 300 成分数据 |
| 未识别标识 | 未识别的数据来源 | 展开后显示原始技术标识 |

推荐显示：

```text
数据来源
新浪财经行情
通过 AKShare 采集 · 查看技术信息
```

不推荐继续直接显示：

```text
sina_via_akshare
```

### 2.2 实现要求

- 前端使用统一的来源展示字典，详情页、列表页和系统状态页共用同一套名称。
- API 继续保留原始 `source` 字段用于审计，不要把中文名称写入历史数据记录。
- 新出现的标识必须显示为“未识别的数据来源”，同时保留可展开的原始值，避免错误翻译。
- 数据来源旁增加“数据类型”，例如“历史日线”“5 分钟行情”“实时行情”，避免用户只看到来源却不知道数据用途。

## 3. 让折线图能够被新手读懂

### 3.1 修改图表标题

当前标题“历史行情”过于宽泛。建议根据周期动态显示：

- 日线：`近 1 年股价走势（日线收盘价）`
- 5 分钟线：`近 5 个交易日走势（每 5 分钟收盘价）`

标题下增加固定说明：

> 横轴是日期，纵轴是价格（元）。折线上的每个点代表一个交易日的收盘价，不是当前实时价格。

切换到 5 分钟线时改为：

> 横轴是交易时间，纵轴是价格（元）。每个点代表一个 5 分钟时段结束时的价格。

### 3.2 图表上方增加“先看结论”

对新手来说，先给出事实摘要，再展示完整折线图更容易理解。建议增加四张小卡：

| 卡片 | 示例文案 | 说明 |
|---|---|---|
| 最新历史收盘 | 3.05 元 | 明确这是最近一个交易日的收盘价 |
| 区间变化 | 近 1 年下降 57.4% | 使用区间首尾收盘价计算，只陈述历史事实 |
| 区间最高 | 7.16 元 | 显示发生日期 |
| 区间最低 | 2.95 元 | 显示发生日期 |

摘要下方增加一句自然语言：

> 过去一年整体呈下降走势。以上仅是历史价格变化，不代表未来表现。

### 3.3 补齐图表阅读元素

- 纵轴明确标注单位“价格（元）”。
- 横轴显示更易读的日期，例如“2025-07”“2026-01”“2026-07”。
- 鼠标移动到折线点时显示：日期、收盘价、较前一交易日涨跌额和涨跌幅。
- 增加“1个月 / 3个月 / 6个月 / 1年 / 3年”时间范围按钮。
- 图表下方显示数据条数和最后更新时间，例如“共 240 个交易日，更新至 2026-07-14”。
- 图表加载、无数据、请求失败分别使用不同文案，不显示空白图框。
- 继续使用中性的蓝色折线；涨跌信息用文字和数值表达，不能只依赖颜色。

### 3.4 日线与 5 分钟线的通俗说明

在周期按钮旁增加问号提示：

| 术语 | 新手说明 |
|---|---|
| 日线 | 每个交易日只保留一个收盘价格，适合观察几个月到几年的趋势 |
| 5 分钟线 | 每 5 分钟记录一次价格，适合观察最近几个交易日的短期波动 |

## 4. 解释“前复权”

### 4.1 主界面文案

将当前的：

```text
复权口径
前复权
```

改为：

```text
价格处理
前复权（便于比较长期涨跌）  [?]
```

问号提示内容：

> 公司分红、送股后，股价会产生机械变化。前复权会调整过去的价格，让不同日期更容易连续比较。图中的历史价格经过计算，可能与当时实际成交价不同。

### 4.2 分阶段实现

- 第一阶段只增加中文解释，不改变现有数据口径。
- 第二阶段只有在 API 确实支持不同口径后，才增加“前复权 / 原始价格”切换。
- 页面必须始终显示当前口径，导出数据时也要携带该字段。

## 5. 增加“获取最新行情”按钮

### 5.1 空状态设计

当实时行情为空时，建议显示：

```text
暂无实时行情
当前还没有取得这只股票的最新报价，历史行情仍可正常查看。

[ 获取最新行情 ]
预计约 5–15 秒，高峰期可能需要 30 秒
```

预计时间是用户提示，不是完成保证。超过 30 秒后停止等待，并明确显示结果。

### 5.2 点击后的完整状态

| 状态 | 按钮文案 | 页面说明 |
|---|---|---|
| 可操作 | 获取最新行情 | 预计约 5–15 秒 |
| 已提交 | 已提交，等待获取… | 显示排队时间 |
| 获取中 | 正在获取…已等待 6 秒 | 按钮禁用，避免重复提交 |
| 成功 | 已更新 | 显示行情时间和“刚刚更新” |
| 失败 | 再试一次 | 显示失败来源、失败时间和简短原因 |
| 非交易时段 | 获取最近可用行情 | 说明可能只能取得最近交易日数据 |
| 请求过于频繁 | 30 秒后可再次获取 | 显示倒计时 |

失败示例：

> 新浪财经行情暂时无法连接（10:35:12）。历史行情不受影响，可稍后再试。

### 5.3 接口建议

建议新增单只股票的刷新接口，不触发全部 300 只股票任务：

```http
POST /api/v1/stocks/{symbol}/quote-refresh
```

推荐首次响应：

```json
{
  "data": {
    "job": {
      "id": "2c69ea4f-3b19-49df-a925-9c60181996ef",
      "job_type": "quote_refresh",
      "symbol": "000002",
      "status": "queued"
    },
    "estimated_seconds": 10,
    "retry_after_seconds": 0,
    "source": "eastmoney_via_akshare",
    "requested_at": "2026-07-15T03:14:55.750193Z"
  }
}
```

然后通过任务状态接口查询：

```http
GET /api/v1/jobs/{job_id}
```

实现要求：

- 一次只请求当前股票，不能因为用户点击而刷新全部自选股。
- 同一股票 30 秒内的重复请求合并为同一任务。
- 按钮只能调用该数据类型预先配置的明确来源。
- 请求失败时返回真实错误状态，历史收盘价不得填入实时行情区域。
- 前端最多等待 30 秒；页面关闭后，任务仍可正常结束并写入行情记录。
- 成功后只刷新实时行情模块，不重新加载公告、预测和完整历史图。

### 5.4 时间标注规则

实时行情区域至少同时展示三个时间：

| 时间 | 示例 | 用途 |
|---|---|---|
| 行情时间 | 2026-07-15 10:34:57 | 上游行情对应时间 |
| 获取时间 | 2026-07-15 10:35:03 | 本系统实际取得数据的时间 |
| 数据年龄 | 约 6 秒前 | 帮助用户判断是否仍然新鲜 |

如果上游没有提供可靠的行情时间，必须写明“行情时间未提供”，只展示系统获取时间。

## 6. 建议页面结构

股票详情首屏建议调整为以下顺序：

```text
股票名称 + 股票代码 + 当前交易状态

实时行情
├─ 最新价 / 涨跌额 / 涨跌幅
├─ 行情时间 / 获取时间 / 数据年龄
└─ 无数据时显示“获取最新行情”按钮与预计时间

历史走势
├─ 先看结论：最新历史收盘 / 区间涨跌 / 最高 / 最低
├─ 日线 / 5 分钟线及时间范围
├─ 新手阅读说明
├─ 折线图与悬浮信息
└─ 中文来源 / 价格处理说明 / 更新截止日期
```

实时行情和历史走势在视觉上使用两个独立卡片，标题和底色略有区分，防止用户把历史收盘价理解为当前价格。

## 8. 验收标准

- 股票详情主界面不直接暴露 `sina_via_akshare`、`eastmoney_via_akshare` 等内部标识。
- 用户无需外部搜索即可理解“日线”“5 分钟线”和“前复权”的基本含义。
- 折线图明确显示时间范围、价格单位、数据截止日期和数据条数。
- 实时行情为空时存在唯一明确的“获取最新行情”按钮，并显示预计 5–15 秒、高峰期可能 30 秒。
- 点击后能看到排队、获取中、成功或失败的真实状态；不能一直停留在无反馈的加载状态。
- 成功后展示行情时间、系统获取时间和数据年龄。
- 失败后明确显示来源、失败时间和原因，历史行情继续可查看。
- 多次快速点击不会创建多个相同任务，也不会触发 300 只股票的整批刷新。
- 实时行情与历史行情继续保持独立，任何区域都不能用另一类价格填补空值。

## 9. 实施与验收结果

> 实施状态：已完成并部署到本地环境
> 验收页面：`http://127.0.0.1:3000/stocks/000002`
> 验收日期：2026-07-15

### 9.1 已完成内容

- 建立统一来源展示字典；行情、系统状态和公告列表使用中文来源名称，内部标识只保留在技术信息中。
- 历史图表改用 ECharts 5.6 模块化 SVG 渲染，仅引入折线图、坐标轴、提示框、无障碍和 SVG 所需模块。
- 图表支持 1 个月、3 个月、6 个月、1 年、3 年区间，标题、条数、截止日期和摘要会随区间同步变化。
- ECharts 悬浮提示显示日期、收盘价、较上一点涨跌额和涨跌幅；纵轴明确使用“价格（元）”。
- 增加“日线”“5 分钟线”“前复权”的通俗解释，并明确历史价格不是当前实时价格。
- 新增单只股票刷新接口 `POST /api/v1/stocks/{symbol}/quote-refresh` 和任务查询接口 `GET /api/v1/jobs/{job_id}`。
- Worker 每次只向固定来源请求当前股票；同一股票 30 秒内合并任务，前端最多等待 30 秒。
- API 返回 1 个月、3 个月、6 个月、1 年、3 年的首尾、涨跌、最高、最低、日期、条数和更新时间摘要。
- 成功后只更新实时行情模块；失败时显示中文来源、失败时间和真实原因，不使用历史收盘价填充实时行情。

- 前端 118 项测试全部通过，覆盖成功、超时、上游失败、限频、休市、来源中文化和 ECharts 标识。

### 9.3 真实页面验收

- ECharts 容器实际生成 SVG，默认展示 243 个近 1 年日线点；切换 1 个月后正确展示 21 个点。
- 悬浮示例正确显示：`2025/12/31`、收盘价 `4.65 元`、较上一点 `+0.03 元`、涨跌幅 `+0.65%`。
- 中文来源正确显示“新浪财经行情”“巨潮资讯”，新闻媒体保留“澎湃新闻”等原名称。
- 午间休市时按钮显示“获取最近可用行情”，并提示预计 5–15 秒、高峰期可能 30 秒。
- 实测刷新任务先显示排队状态，随后真实返回东方财富行情服务 HTTP 500；页面明确显示失败来源、时间和原因，并说明历史行情不受影响，30 秒内不会重复创建任务。

当前外部行情服务仍可能返回 HTTP 500，这属于上游可用性问题；本次改造已完成用户可操作入口、状态反馈、限频和历史/实时数据隔离，不会把失败伪装成成功。
