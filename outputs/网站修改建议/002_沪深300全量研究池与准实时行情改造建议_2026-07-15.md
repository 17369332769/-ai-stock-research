## 1. 需求定义

系统不再把“沪深300成分股”和“用户自选股”混为同一个概念，而是形成两层研究范围：

```text
系统默认研究池
└─ 当前沪深300全部成分股，自动维护、全部分析

额外自选
└─ 用户主动添加的非沪深300股票

实际跟踪范围
└─ 当前沪深300 ∪ 额外自选，按股票代码去重
```

产品规则：

- 当前沪深300的300只股票自动出现在首页，无需用户逐只添加。
- 沪深300中的每只股票都采集行情、历史K线、公告新闻并执行研究分析。
- “添加自选”只用于非沪深300股票；沪深300股票不显示重复添加操作。
- 非沪深300股票加入额外自选后，进入与沪深300相同的数据回补和研究流程。
- 股票调入或调出沪深300时自动调整跟踪状态，但历史数据不得删除。

## 2. 当前系统基线

改造前截至2026-07-15，本地数据库状态为：

| 数据 | 当前数量 | 时间范围或说明 |
|---|---:|---|
| 实时报价 `quotes` | 0条 | 当前没有任何股票实时价格 |
| 日线 | 215,806条 / 300只 | 2023-07-17至2026-07-14 |
| 5分钟线 | 43,563条 / 300只 | 2026-07-10至2026-07-15 |
| `000538` 日线 | 725条 | 约3年 |
| `000538` 5分钟线 | 160条 | 最近数个交易日 |
| 模型版本 | 0个 | 尚未训练并注册可用模型 |

当前报价 Provider 虽然接收股票代码列表，但内部仍调用 `stock_zh_a_spot_em()` 获取整个沪深京A股市场，再按代码过滤。现有12秒缓存缓存的也是全市场快照，因此大接口失败会造成全部关注股票都没有报价。

当前主要问题：

1. 全市场免费接口体积大、耗时长且容易失败。
2. 300只沪深300股票被直接写成普通自选，无法区分系统研究池和用户额外关注。
3. 实时报价表为空，详情页只能展示历史行情。
4. 日线训练数据已经存在，但模型训练与注册尚未完成。
5. 5分钟线只有最近数日，不足以训练可靠的盘中模型。

## 3. 改造后的产品效果

### 3.1 首页

首页默认显示三个入口：

```text
沪深300研究池（300）
额外自选（用户添加）
全部关注（合并去重）
```

沪深300列表每行展示：

- 股票代码和名称；
- 最新价、涨跌额、涨跌幅；
- 今开、最高、最低、昨收；
- 成交量、成交额；
- 行情时间、系统获取时间、数据年龄；
- 最新、延迟、可能过期、获取失败等状态；
- 今日异动、公告事件和分析状态。

首页顶部增加市场摘要：

```text
今日上涨 186只    今日下跌 107只
行情异常 12只     有重要公告 8只
数据延迟 3只      等待分析 5只
```

### 3.2 股票详情

股票详情页正常情况下自动获得当前报价，不再要求用户先点击按钮：

```text
云南白药 000538
沪深300成分股 · 下午交易中

最新价      涨跌额      涨跌幅
今开        最高        最低        昨收
成交量      成交额      换手率

行情时间：14:32:15
系统获取：14:32:18
约3秒前更新
```

ECharts历史行情继续独立展示，不允许使用历史收盘价填充实时价格。只有报价过期或失败时才显示“重新获取”按钮。

### 3.3 额外自选

用户搜索非沪深300股票后可以加入额外自选。加入后自动执行：

1. 回补至少3年日线；
2. 回补上游可提供的5分钟线；
3. 获取公告和新闻；
4. 加入行情刷新；
5. 生成特征、分析和预测；
6. 标注“非当前沪深300成分股”。

## 4. 股票集合与生命周期

### 4.1 集合定义

| 集合 | 来源 | 是否自动维护 | 是否参与分析 |
|---|---|---|---|
| 沪深300研究池 | 指数成分历史 | 是 | 是，全部参与 |
| 额外自选 | 用户操作 | 否 | 是 |
| 当前查看股票 | 页面会话 | 临时 | 只提高行情优先级 |

### 4.2 成分变更

- 新调入沪深300：自动进入研究池并触发数据回补。
- 调出沪深300：停止系统级主动刷新，但保留全部历史数据。
- 调出股票仍在额外自选：继续刷新和分析。
- 额外自选后来调入沪深300：自动去重，转为沪深300来源。
- 历史训练必须按每个交易日当时的成分股构建，禁止用今天的300只股票回填历史。

### 4.3 现有自选数据迁移

当前300只系统初始化股票需要从普通自选中迁出：

1. 备份现有 `watchlist_items`；
2. 将当前沪深300成员从用户自选记录中移除；
3. 保留不属于当前沪深300的记录作为“额外自选”；
4. 首页接口返回“沪深300研究池 ∪ 额外自选”；
5. 迁移过程不得删除任何行情、K线、公告、分析或预测记录。

## 5. 准实时行情方案

### 5.1 主链路

300只股票全部达到15至30秒刷新，主链路必须使用“按股票代码批量查询”的行情服务：

```text
跟踪股票集合
    ↓
后端批量查询指定股票代码
    ↓
校验、标准化、去重
    ↓
UPSERT latest_quotes
    ↓
网页读取本地API
```

硬性要求：

- 上游只能返回请求的股票集合，不能下载整个A股市场再过滤。
- 浏览器不能直接访问外部行情地址。
- 多个页面打开时仍由后端统一刷新，不能成倍增加上游请求。
- 同一批次部分股票失败时，只标记对应股票，不丢弃其他成功报价。

### 5.2 单股票接口

单股接口用于：

- 当前正在查看的股票；
- 新加入的额外自选；
- 批量结果缺失时的有限补偿；
- 用户主动点击“重新获取”。

AKShare `stock_bid_ask_em(symbol)` 可返回最新价、涨跌、成交量、成交额、换手率、量比、最高、最低、今开、昨收以及买一卖一等盘口字段。该接口没有可靠交易所行情时间时，页面必须显示“行情时间未提供”，不得把系统获取时间冒充行情时间。

### 5.3 刷新频率

| 对象 | 有正式批量服务 | 仅有免费单股服务 |
|---|---:|---:|
| 沪深300全部报价 | 15至30秒 | 2至5分钟轮询 |
| 当前查看股票 | 15秒 | 15至30秒 |
| 额外自选 | 15至30秒 | 30至60秒，设置数量上限 |
| 休市时段 | 停止主动刷新 | 停止主动刷新 |

若坚持让300只股票每15秒逐股调用单股票接口，将产生约20次请求/秒、28.8万次请求/交易日，不适合作为免费数据源的长期方案。

### 5.4 缓存、并发与熔断

- 每个股票代码保留12秒报价缓存。
- 相同股票的并发请求合并为同一个上游任务。
- 全局设置并发上限，避免瞬时请求风暴。
- 同一来源连续失败3次后进入5分钟冷却。
- 冷却恢复后先使用少量探测请求，成功后再恢复正常流量。
- 可采用1分钟、5分钟、15分钟的递增冷却，并增加随机抖动。
- 定时刷新、详情页刷新和手动刷新必须共享同一份熔断状态。

### 5.5 新鲜度状态

交易时段建议采用：

| 数据年龄 | 页面状态 |
|---:|---|
| 0至45秒 | 最新 |
| 46至120秒 | 行情可能延迟 |
| 121至180秒 | 行情可能已过期 |
| 超过180秒或从未成功 | 暂无可用行情 |

休市时不使用相同的秒级过期语义，而是显示“最近交易时段行情”和对应日期。

## 6. 页面与交互设计

### 6.1 首页筛选与排序

支持按以下条件筛选：

- 沪深300 / 额外自选 / 全部关注；
- 上涨 / 下跌 / 平盘；
- 行情最新 / 延迟 / 失败；
- 有异动 / 有公告 / 有预测；
- 行业分类。

支持按涨跌幅、成交额、数据年龄、异动强度和分析更新时间排序。

### 6.2 状态文案

正常：

> 东方财富行情 · 14:32:15行情 · 约3秒前获取

延迟：

> 行情可能延迟，最后成功获取于14:30:02。

失败：

> 行情服务连续失败，已暂停请求5分钟。历史行情和已有分析仍可正常查看。

无上游行情时间：

> 行情时间未提供 · 系统获取于14:32:18

## 7. 分析任务设计

### 7.1 盘中任务

| 任务 | 范围 | 建议频率 |
|---|---|---:|
| 最新报价 | 沪深300＋额外自选 | 15至30秒或降级频率 |
| 5分钟K线 | 全部研究范围 | 每5分钟 |
| 量价异动检测 | 沪深300全部 | 每5分钟 |
| 相对强弱 | 沪深300全部 | 每5分钟 |
| 异动事件检索 | 触发异动的股票 | 按需 |

### 7.2 收盘后任务

- 更新全部股票日线；
- 对账开高低收、成交量和成交额；
- 构建每日特征；
- 生成预测和置信度；
- 结算历史预测；
- 更新模型成绩与漂移状态；
- 生成沪深300每日研究摘要。

### 7.3 额外自选分析

额外自选可复用沪深300模型推理，但必须显示：

> 该股票不属于模型主要训练股票池，预测可靠性可能较低。

如果未来积累了更广泛的股票历史，应单独训练“全A股模型”或按行业训练模型，不能静默扩大现有模型适用范围。

## 8. 训练数据方案

### 8.1 数据职责分离

| 数据 | 主要用途 | 是否直接作为长期训练历史 |
|---|---|---|
| `latest_quotes` | 页面展示、当前推理参考 | 否 |
| 短期报价快照 | 排障、聚合分钟线 | 默认否 |
| 5分钟K线 | 盘中特征、盘中模型 | 是，但需更长历史 |
| 日线 | 日频特征、训练、回测 | 是 |
| 公告新闻 | 事件特征 | 是 |
| 历史成分 | 构建无幸存者偏差样本 | 是 |

### 8.2 当前数据可用性

- 约3年日线已经覆盖300只股票，可用于第一版日频模型的数据质量检查和走步训练。
- 5分钟线只有最近约5个交易日，不足以训练可靠的盘中模型。
- 若要训练盘中模型，需要持续积累数月以上5分钟线，或接入可靠历史分钟数据服务。
- 实时报价只能作为当前推理参考，不能用几天的报价快照冒充长期训练数据。

### 8.3 训练实施要求

1. 检查日线缺失、停牌、复权、成交量单位和异常值；
2. 使用当时有效的沪深300成员构建样本；
3. 按时间进行训练、验证、测试和走步回测；
4. 保留简单基准模型；
5. 候选模型未优于基准时不得激活；
6. 模型产物写入版本目录并登记 `model_versions`；
7. 页面只读取 `active` 模型；没有可用模型时继续明确显示“模型不可用”。

## 9. 数据模型调整

### 9.1 `latest_quotes`

建议新增一行一股票的最新报价表：

| 字段 | 说明 |
|---|---|
| `symbol` | 股票代码，主键 |
| `price` | 最新价 |
| `change_amount` | 涨跌额 |
| `change_percent` | 涨跌幅 |
| `open` / `high` / `low` | 今开、最高、最低 |
| `previous_close` | 昨收 |
| `volume` / `amount` | 成交量、成交额 |
| `turnover_rate` | 换手率，可空 |
| `bid1` / `ask1` | 买一价、卖一价，可空 |
| `market_time` | 上游行情时间，可空 |
| `fetched_at` | 系统获取时间 |
| `source` / `source_url` | 来源与审计地址 |
| `raw_payload` | 原始上游响应 |

### 9.2 短期报价历史

如需保留盘中快照，建议将现有 `quotes` 明确为短期 `quote_ticks`：

- 只保留7至30天；
- 定期聚合成1分钟或5分钟K线；
- 设置自动清理任务；
- 不影响永久保存的日线和正式分钟线。

如果当前不训练盘中模型，可暂不保存全部15秒快照，只保存最新报价与必要审计记录，避免数据库无意义增长。

### 9.3 集合接口

建议提供：

```http
GET    /api/v1/research-pool?scope=csi300|extra|all
POST   /api/v1/extra-watchlist
DELETE /api/v1/extra-watchlist/{symbol}
GET    /api/v1/quotes/latest?scope=all
POST   /api/v1/stocks/{symbol}/quote-refresh
```

## 11. 验收标准

### 11.1 研究池

- 首页自动展示当前沪深300全部300只股票。
- 用户无需把沪深300股票手动加入自选。
- 额外自选只保存非沪深300股票。
- 沪深300与额外自选合并后无重复代码。
- 成分调出后历史数据保留，额外自选状态不丢失。

### 11.2 行情

- 交易时段300只股票均能展示最新报价或明确失败状态。
- 主链路不再调用 `stock_zh_a_spot_em()` 获取整个A股市场。
- 当前股票可通过单股接口独立刷新。
- 相同股票12秒内的并发请求只触发一次上游调用。
- 连续失败后自动冷却，不持续冲击上游。
- 页面同时显示行情时间、获取时间和数据年龄；行情时间缺失时明确说明。
- 过期报价不得标记为实时。
- 历史收盘价不得填入实时报价区域。

### 11.3 分析与训练

- 沪深300全部股票每日都有数据完整性与分析状态。
- 训练样本使用历史时点的成分关系，不存在当前成分回填历史。
- 日频模型完成训练、验证、基准比较和版本登记。
- 5分钟历史不足时不得宣称盘中模型可用。
- 额外自选使用沪深300模型时显示适用范围提示。

## 12. 风险与边界

1. 免费单股接口无法稳定支撑300只股票15秒刷新；正式目标依赖可靠批量行情服务。
2. AKShare和东方财富页面接口不是交易所级行情，不应宣传为毫秒实时数据。
3. 上游未提供行情时间时，只能展示系统获取时间。
4. 5分钟历史数据不足是盘中训练的主要限制，不能用实时快照简单替代。
5. 沪深300成分历史必须长期保存，否则模型会产生幸存者偏差。
6. 本系统继续只做研究与复盘，不提供交易功能，也不承诺收益。
